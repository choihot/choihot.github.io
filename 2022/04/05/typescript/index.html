TypeScript学习

TypeScript环境配置

  • 电脑全局安装：npm install -g typescript
    • 检查是否安装和安装的版本 tsc -v
  • 也需要安装Node.js

TypeScript运行

  • 新建一个main.ts文件，输入:
  • 在terminal中将.ts文件编译成.js文件
    • tsc main.ts
  • 然后运行
    • node main.js
  • 就可以在teminal中看到”hello world”了
    • 为什么.js文件没有编译就能运行？
    • 因为NodeJS原生支持TypeScript。
      • 可以直接用node main.ts运行.ts文件

TypeScript工作流

  • 写好ts代码以后，要用ts的编译器把.ts文件编译为.js文件，然后又浏览器调用并且执行.js文件。
  • ts的编译器就叫做TSC。
  • 为什么需要TSC编译？
    • 因为大部分主流浏览器还不支持TypeScript.

TypeScript demo项目依赖

  1. npm init
  2. npm i --save-dev lite-server
  • lite-server是用来开发的轻量级服务器。
  • package.json中的”devDependencies”和”dependencies”区别
    • devDependencies是参与项目开发，项目开发完成上线并不需要打包部署到服务器中。
    • dependencies下所有的依赖都会被打包放到服务器上。
    • 区分的好处：有效降低网站大小，缩短用户打 开网站的时间。
  1. 添加”start”: “lite-server”到package.json中的script下。
    1. 可以自动部署在localhost:3000。

TypeScript 基础

声明变量

  • 通常用let。
    • 使用var的坏处：
      • 只有全局作用域和函数作用域，没有块级作用域。而且可以多次声明相同的变量。
    • let就是用来解决var所带来的的问题。
      function doSomething(){
  for (var i=0; i< 5; i++){
    console.log(i);
  }
// 跳出for循环之后，i还是存在。i=5。
// 使用let可以解决这个问题，会立马报错。
  console.log("finally i = ", i);
}
doSomething()

TypeScript的基本类型

  • 常见的：boolean, string, number, array, null, undefined, object.

  • 不常见的：tuple, enum, void, never, any.

  • 常见高级类型：

    • Union 联合类型
    • Nullable 可空类型
    • Literal 预定义类型

number, string, boolean

number
  • 既能表示整数，也能表示浮点数或者正负数。
string
  • “hello”, ‘hello’, `hello`
  • 反引号：``，可以创建一个字符串模板。
noolean 布尔类型
  • 真、假
  • 处理逻辑判断
使用例子

  return n1 + n2;

array, tuple

array 数组类型
  • []
  • 可以存放任意类型的数据。
    // 标准写法
let list1: number[] = [1,2,3,4];
// 泛型
let list2: Array<number> = [1,2,3,4];
// 不写类型的偷懒写法
let list3 = [1,2,3,4];

// 显示类型：let list4: (string | number)[]
let list4 = [1, "ddd"];
// 显示类型: let list5: any
let list5: any[] = [1, "ddd", true];
// 不同：list4只能存放string和number， list5存放任何类型的数据。
tuple 元祖类型
  • 特殊的数组。是固定长度，固定类型的数组。
  • 只有声明时同时指定了变量类型才是tuple元祖。
    • 没有指定类型就是普通的array。
      let person1: [number, string] = [1, "ddd"]

///下面都会报错
person1[0] = "ddd"
person1[1] = 1
person1[2] = 111
  • 需要注意：元祖现在还有bug。
    • push()可以无限拓宽元祖的长度。
      person1[2] = 3 //会报错，但如果利用push()则不会报错，还能通过编译。
person1.push(3)

Any, Unknown

  • any是一个动态类型，支持和兼容所有类型

  • 在不知道用什么类型的时候可以用any。

  • 在日常使用js的时候就是个any类型。

  • js的灵活性带来的快速开发效率也给代码带来了不可阅读性，不可维护性，不可拓展性等一系列的问题。

  • any类型保留了高灵活性同时带来了强类型语言才有的高可维护性。

    let randonValue: any = 666;
randomValue = true;
randomValue = "dddd";
randomValue = {};
randomValue(); //编译执行到这里的时候会报错。
randomValue.toUpperCase()

  • unknown和any差不多，不能确定变量的类型，但是可以保证类型的安全。

    用法：将上面的代码略作修改
if(typeof randomValue === 'function'){
  randomValue();
}
if(typeof randomValue === 'string'){
  randomValue.toUpperCase();
}

  • 如何选择？

    • any适合代码的快速成型快速上线，但是会遗留下来一些安全隐患。
    • unknown就更保险一点。

Void, Undefined, Never

  • Void:

    • Void和Undefined区别就在于在探讨某个变量到底是否存在的哲学问题。
    • Undefined指的是变量没有赋值没有初始化。
    • Void指的是变量本身就不存在。
    • ts中，所有的void输出的都是undefined。
      function printResult(): void {
  console.log("lalala")
}
console.log(printResult(()) //输出undefined

function printResult(): undefined {
  console.log("lalala")
  return //一定要加return,否则会报错  
}
console.log(printResult(()) //输出undefined

  • Never

    • 一个函数永远执行不完，就是Never的本质。
    • 用法：用来处理异常或者Promise
    • 下面的函数本质上没有返回值。返回值的类型会显示”void”。
    • 下面这个函数不仅没有返回值，还会抛出异常。也就是执行到”throw”的时候就会强行让该函数结束了。永远不会执行完成。
      function throwError(message: string, errorCode: number): never{
  throw{
    message,
    errorCode
  }
}
throwError("not found", 404)
    • 除了抛出异常以外，函数无法执行完成的方法：使用while循环。
      function whileLoop(): never{
  while(true){
    console.log("dddd")
  }
}

TypeScript的面对对象

Object 对象类型

  • js中使用花括号并且以键值类形式实现的数据，都属于对象类型。

  • 但是ts更明确来说是key to type，键类型对。

    const person = {
  firstName : "choi",
  lastName: "hot",
  age: 18
}
console.log(person.nickname)//调用没有定义的内部变量，js不会报错，但是ts会。

//如果不怕麻烦可以这样定义：
const person = {
  firstName : string,
  lastName: string,
  age: number
} = {
  firstName : "choi",
  lastName: "hot",
  age: 18
}

<!-- 
  笼统定义object类型不会对代码有任何帮助，ts实际上不知道对象内部结构。
  指定了object类型相当于指定了一对空的花括号。
  const person: object 相当于 const person: {} 
  ts知道它是一个对象，但是不知道内部情况。  
-->

const person: object = {
  firstName : "choi",
  lastName: "hot",
  age: 18
}
console.log(person.firstName)//会报错

    Interface, Class

    • Interface面对对象接口：
    • 接口就相当于一份说明书，类的实现都要按照这份说明书来。
      let drawPoint = (point) => {
  console.log({x: point.x, y:point.y});
}

//这个时候没有对point进行类型限制，所以可以随便传参，会导致代码和业务逻辑都出错。
//后面两行代码并不会报错。
drawPoint({x:105,y:24});
drawPoint({x:"choi", y:"hot"});
drawPoint({weather: "干燥", temperature:"5oC"});

//所以要创建interface接口对point进行类型限制。
    interface Point{
      x: number;
      y: number;
    }

//将最上面的代码改为：
let drawPoint = (point：Point) => {
  console.log({x: point.x, y:point.y});
}
//这时候刚才没报错的两行代码就会报错了。
    • 因为有对象就会引出了“高内聚，低耦合”的问题。上面的代码中的drawPoint和Point这个对象高度相关，应该用面对对象的思维来解决，不然写的代码都是低内聚的。
    • 把所有相关的属性和方法都打包在一个集合中，就是Class。

    • Class类

    • 对class的理解：

      • 封装：对具体细节具体过程的隐藏。

    • 重写上面的drawPoint代码:

      interface IPoint{
  x: number;
  y: number;
  drawPoint:()=>void; //打印坐标点信息
  getDistances: (p:IPoint) => number; //需要传入对方坐标点信息
}

//需要实现IPoint中所有的成员变量和成员方法。
class Point implements IPoint{
  x: number; 
  y: number;
  drawPoint = () =>{
    console.log("x: ", this.x, " y: ", this.y);
  }
  getDistances = (p: IPoint) =>{
    return Math.pow(p.x - this.x, 2) + Math.pow(p.y-this.y,2)
  }
}

const point = new Point();
<!-- point.x = 2;
point.y = 3; -->
point.drawPoint()

      • 类class就是Point，对象就是这个class Point 的实例。也就是说使用new实例化以后就产生了一个对象。对象也被称作实例，instance。

      • 运行drawPoint的时候报错，因为x,y没有初始化。可以直接赋值。

        • 但更好的方法是使用构造函数 constructor。
        • 改写上述代码：
          class Point implements IPoint{
x: number; 
y: number;

constructor(x: number, y: number){
  this.x = x;
  this.y = y;
}

drawPoint = () =>{
  console.log("x: ", this.x, " y: ", this.y);
}
getDistances = (p: IPoint) =>{
  return Math.pow(p.x - this.x, 2) + Math.pow(p.y-this.y,2)
  }
}

const point = new Point(2,3); //通过构造函数，在Point对象初始化的时候就可以给x,y赋值了。
        • 那么如果在实例化的时候并不知道具体需要赋值多少，或者不想赋值怎么办？
          • 加一个问号 ? ，变成可选参数。
          • 或者给参数一个默认值。
                      constructor(x?: number, y: number = 2){
            this.x = x;
            this.y = y;
          }
        ```     
      - 注意：JS构造函数不可以重载，一个类有且仅有一个constructor。
      - 上面的赋值方法较为繁琐，于是引入了接下来的访问修饰符的概念。

#### Access Modifier
  - 访问修饰符分为public, private, protected.
  - 一般默认的成员变量都是public

      //利用public改写上面的代码:
      class Point implements IPoint{
      //不需要在这里声明x,y的类型。
      //不能再使用”?”做可选参数。
      constructor(public x: number, public y: number = 2){

      //不需要再用this.x = x进行赋值。

      }
      }

      const point = new Point(24, 50);
      //但是public方法下，依旧可以重新赋值，在实际工作中是非常危险的。
      point.x = 30;
      point.y =5;

      
- 所以可以使用private，把公有变成私有，就能让外部无法访问。

      class Point implements IPoint{

      constructor(private x: number, private y: number = 2){}

      }
      //尝试使用point.x = 30进行修改则会报错。

      
- 但是又会出现新的问题，private会导致和IPoint接口声明的成员不一致。**接口的成员变量都默认为public**。设置为private会导致Point类报错。
  - 最直接的解决方式是**在IPoint接口中直接删除x,y这两个公有变量。**
  - 考虑到有从外部访问成员变量的需要，要用**getter，setter**这两个方法。比如getDistances函数就需要访问x,y。
    - getter,setter: 把class的私有属性封装一下，让外部通过特定的方法(getter,setter)进行访问。

      getX = () =>{

      console.log(point.getX());

      //同理可以对纵坐标进行getter，setter的设置。
      setY = (value: number) =>{…}

#### Generic
- **Generic泛型**
- 考虑到**可重用性**，不仅支持当前的数据类型，也支持未来的数据类型而存在。
- 使用一个identity函数做例子:
  - 不用泛型的话:`` function identity(arg: number): number {
    return arg;
    } ``
  - 使用Any的话:`` function identity(arg: any): any {
    return arg;
    } `` 
  - 但是使用Any我们只能知道有任意类型的数据被传入和返回。**并不能绑定和追踪两个值的类型。**
  - 所以使用泛型: `` function identity<T>(arg: T): T {
    return arg;
    } ``
- **泛型是可以帮助捕获用户传入的类型，同时使用这个类型当做返回值的类型。**
- 同样是声明数字数组: 
  - 普通声明：`` let list1: number[] = [1,2,3,4]``
  - 泛型声明：`` let list2: Array<number> = [1,2,3,4]``

- 使用例子:

      //返回数组最后一位元素
      let lastInArray = (arr: T[]) =>{
      return arr[arr.length - 1]
      }
      //会识别l1类型为number
      const l1 = lastInArray([1,2,3,4]);
      //会识别l2类型为string
      const l2 = lastInArray([“a”,”b”,”c”]);
      //因为有指定类型，所以l3的类型为string | number的数组
      const l3 = lastInArray<string | number>([“a”,”b”,”c”]);

      //也可以传入一个默认类型参数
      let makeTuple = <T, Y = number>(x: T, y: Y) = > [x,y]

      //帮我们动态匹配类型，v1识别为 string | number 的元祖类型
      const v1 = makeTuple(1, “one”)
      //v2显式定义了类型，因为默认参数是number，就可以省略第二个参数
      const v2 = makeTuple(true,1);
      ```
