TypeScript学习

TypeScript环境配置

  • 电脑全局安装：npm install -g typescript
    • 检查是否安装和安装的版本 tsc -v
  • 也需要安装Node.js

TypeScript运行

  • 新建一个main.ts文件，输入:
  • 在terminal中将.ts文件编译成.js文件
    • tsc main.ts
  • 然后运行
    • node main.js
  • 就可以在teminal中看到”hello world”了
    • 为什么.js文件没有编译就能运行？
    • 因为NodeJS原生支持TypeScript。
      • 可以直接用node main.ts运行.ts文件

TypeScript工作流

  • 写好ts代码以后，要用ts的编译器把.ts文件编译为.js文件，然后又浏览器调用并且执行.js文件。
  • ts的编译器就叫做TSC。
  • 为什么需要TSC编译？
    • 因为大部分主流浏览器还不支持TypeScript.

TypeScript demo项目依赖

  1. npm init
  2. npm i --save-dev lite-server
  • lite-server是用来开发的轻量级服务器。
  • package.json中的”devDependencies”和”dependencies”区别
    • devDependencies是参与项目开发，项目开发完成上线并不需要打包部署到服务器中。
    • dependencies下所有的依赖都会被打包放到服务器上。
    • 区分的好处：有效降低网站大小，缩短用户打 开网站的时间。
  1. 添加”start”: “lite-server”到package.json中的script下。
    1. 可以自动部署在localhost:3000。

TypeScript 基础

声明变量

  • 通常用let。
    • 使用var的坏处：
      • 只有全局作用域和函数作用域，没有块级作用域。而且可以多次声明相同的变量。
    • let就是用来解决var所带来的的问题。
      function doSomething(){
  for (var i=0; i< 5; i++){
    console.log(i);
  }
// 跳出for循环之后，i还是存在。i=5。
// 使用let可以解决这个问题，会立马报错。
  console.log("finally i = ", i);
}
doSomething()

TypeScript的基本类型

  • 常见的：boolean, string, number, array, null, undefined, object.

  • 不常见的：tuple, enum, void, never, any.

  • 常见高级类型：

    • Union 联合类型
    • Nullable 可空类型
    • Literal 预定义类型

number, string, boolean

number
  • 既能表示整数，也能表示浮点数或者正负数。
string
  • “hello”, ‘hello’, `hello`
  • 反引号：``，可以创建一个字符串模板。
noolean 布尔类型
  • 真、假
  • 处理逻辑判断
使用例子
<!-- 限定传进来的数据类型为number -->
function add(n1: number, n2: number) {
  return n1 + n2;
}
console.log(add(number1, number2));
<!-- 类型限制可以自动映射 -->
let isTrue = true;
isTrue = "true"; //会报错

<!-- 或者手动限制类型 -->
let isTrue: boolean = true;
isTrue = "true"; //会报错

let str: string = "";

array、tuple

array 数组类型
  • []
  • 可以存放任意类型的数据。
    // 标准写法
let list1: number[] = [1,2,3,4];
// 泛型
let list2: Array<number> = [1,2,3,4];
// 不写类型的偷懒写法
let list3 = [1,2,3,4];

// 显示类型：let list4: (string | number)[]
let list4 = [1, "ddd"];
// 显示类型: let list5: any
let list5: any[] = [1, "ddd", true];
// 不同：list4只能存放string和number， list5存放任何类型的数据。
tuple 元祖类型
  • 特殊的数组。是固定长度，固定类型的数组。
  • 只有声明时同时指定了变量类型才是tuple元祖。
    • 没有指定类型就是普通的array。
      let person1: [number, string] = [1, "ddd"]

///下面都会报错
person1[0] = "ddd"
person1[1] = 1
person1[2] = 111
  • 需要注意：元祖现在还有bug。
    • push()可以无限拓宽元祖的长度。
      person1[2] = 3 //会报错，但如果利用push()则不会报错，还能通过编译。
person1.push(3)

Union联合类型与Literal字面量类型

Union联合类型
  • 可以同时支持两个或者几个不同的类型。类型之间用 | 进行分隔，
    let union : string | number 
union = 2;
union = "dddd";


let union2 : number | string | boolean | string[]

function merge(n1: number | string, n2: number | string){
  if(typeof n1 === "string" || typeof n2 === "string“)
    return n1.toString() + n2.toString()
  else
    return n1 + n2
}

let mergeNumber = merge(2,5); //7
let mergeString = merge("hello", "world");//hello world
Literal字面量类型
  • 字面量是 JavaScript 本身提供的一个准确变量。
  • 在声明类型的同时声明值，明确的取值数字的类型。
      //如使用字符串字面量作为一个类型。
  let foo: 'Hello';
  foo = 'Bar'; //会报错
  
  //确定了值只能为数字0,1,2的union。
  let union3: 0 | 1 | 2
  union3 = 1
  //union3 = 4 //会报错

  //改写上面的union2
  let union2 : number | string | boolean | string[]

  function merge(n1: number | string, n2: number | string, resultType: "as-number" | "as-string"){
  //如果resultType是等于"as-string",不管n1和n2是什么类型，让结果输出为string。
  if(resultType === "as-string"){
    return n1.toString() + n2.toString()
  }
  if(typeof n1 === "string" || typeof n2 === "string“)
    return n1.toString() + n2.toString()
  else
    return n1 + n2
}
  let mergeNumber = merge(2,5, "as-number"); //7
  let mergeNumber = merge(2,5, "as-string"); //25
  let mergeString = merge("hello", "world","as-string");//hello world

枚举类型 Enum

enum Color{
  red,
  green,
  blue
} 

let color = Color.blue
console.log(color); // 2
//同理，Color.red = 0, Color.green = 1

//也可以自己定义值
enum Color2{
  red = 5,
  green = 10,
  blue = 1
}

//也可以不是数字，混合string和数字进行处理
enum Color3{
  red = "red",
  green = "green",
  blue = 1
}
let color3 = Color3.green
console.log(color3) //输出"green"

any 和unknown

  • any是一个动态类型，支持和兼容所有类型

  • 在不知道用什么类型的时候可以用any。

  • 在日常使用js的时候就是个any类型。

  • js的灵活性带来的快速开发效率也给代码带来了不可阅读性，不可维护性，不可拓展性等一系列的问题。

  • any类型保留了高灵活性同时带来了强类型语言才有的高可维护性。

    let randonValue: any = 666;
randomValue = true;
randomValue = "dddd";
randomValue = {};
randomValue(); //编译执行到这里的时候会报错。
randomValue.toUpperCase()

  • unknown和any差不多，但是可以保证类型的安全。
