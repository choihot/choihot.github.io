搭建React项目

npx create-react-app my-app 
cd my-app
npm install 
npm start

React设计理念

  • 单向数据流
    • 数据与界面绑定
    • 单向渲染
  • 虚拟dom
    • 如果真实的UI需要更新，对比需要更新的UI，以最小的代价更新UI的元素。
  • 组件化
    • 一个项目是由无数个独立的组件互相搭配组合起来的。
    • 能保持交互一致性
    • 保持视觉风格的统一
    • 便于程序员之间互相协作。

React基础

React函数式组件

JSX

  • React认为视图的本质就是渲染逻辑与UI视图表现的内在统一。

  • 把HTML和渲染逻辑进行了耦合，形成了JSX。

    JSX特点

    1. 常规的HTML代码都可以与JSX兼容。

    2. 在JSX中嵌入表达式, 使用花括号包裹。

      const name = 'Josh Perez';
const element = <h1>Hello, {name}</h1>;
      function formatName(user) {
return user.firstName + ' ' + user.lastName;
}

const user = {
firstName: 'Harper',
lastName: 'Perez'
};

const element = (
  <h1>
    Hello, {formatName(user)}!
  </h1>
);

    3. 使用JSX指定子元素

    • Empty tag can close with />.
    • Can contain children
      const element = <img src={user.avatarUrl} />;

const element = (
  <div>
    <h1>Hello!</h1>
    <h2>Good to see you here.</h2>
  </div>
);

    1. JSX命名约定

    • React Dom使用camelCase(驼峰)方式定义属性

    • 如 HTML 的 class 变成了 className，而 tabindex 则变为 tabIndex


    1. JSX表示对象
    • Babel 会把 JSX 转译成一个名为 React.createElement() 函数调用。
    • 以下两行代码完全等效
      const element = (
  <h1 className="greeting">
    Hello, world!
  </h1>
);

const element = React.createElement(
  'h1',
  {className: 'greeting'},
  'Hello, world!'
);
    • 创建后的对象实际上是这样的：
      // 注意：这是简化过的结构
const element = {
  type: 'h1',
  props: {
    className: 'greeting',
    children: 'Hello, world!'
  }
};
    • 而这些对象都被称为 “React 元素”。

TSX

  • 文件名.tsx。
  • 在配置文件中tsconfig.json启用jsx选项。

配置css模组(module)

  • 架构项目中的样式文件

    • css文件与component文件放在同一个目录下。
    • 命名规范：*.module.css。

  • css module

    • CSS Modules 允许你在不同的文件中使用相同的 CSS classname，而无需担心命名冲突。
    • 每个jsx或者tsx文件都被视为一个独立存在的原件、
    • 原件所包含的所有内容也同样应该独立存在。
    • 组件化角度，简单的import容易造成css的全局污染和冲突。
      • 改进：import './index.css' -> import style from './index.css'
      • 但是这时候ts无法识别css文件，需要给css加上类型的定义声明。
      • TS的定义声明
        • *.d.ts文件
        • 只包含类型声明，不包含逻辑。
        • 不会被编译，也不会被webpack打包。
        • 创建一个custom.d.ts文件
          declare module "*.css" {
const css: {[key: string]: string};
export default css;
}
      • css引入方法
        1. 直接引入整个css文件
          import './index.css'
<div className="app" />
        2. JSS模块化引入组件
          import styles from './index.css'
<div className={styles.app}>

  • css与TS

    • 安装该插件到开发环境中。
      • npm install typescript-plugin-css-modules --save-dev
    • 添加到tsconfig.json中
      • "plugins": [{ "name": "typescript-plugin-css-modules" }]
    • 使用vscode的话还需要在项目根目录下创建.vscode文件夹
      • 并在该目录下创建settings.json
        {
  "typescript.tsdk": "node_modules/typescript/lib",
  "typescript.enablePromptUseWorkspaceTsdk": true
}

  • 对字体的引用

    • 使用@font-face在css中定义名字和字体文件位置。
      @font-face {
  font-family: 'Roboto';
  src: local('Roboto') , url(./assets/fonts/Roboto-Regular.ttf);
}
    • 直接使用定义后的名字：
      h1 {
  font-family: 'Roboto';
  font-size: 72px;
}

React class类组件

  • class xxx extends React.Component<P = {}, S = {}, SS = any> 有三个状态.
    • P: Props
      • 组件间传递数据
    • S: State
      • 组件自己的状态
    • SS: 自定义数据
      • 在生命周期getSnapshotBeforeUpdate()中使用，可以在UI渲染之前获得一些相应的DOM数据。
  • 用render()函数返回react元素。
    • ShoppingCart例子
      import React from 'react';
import styles from './ShoppingCart.module.css';
interface Props {

}

interface State {
    isOpen: boolean;
}
class ShoppingCart extends React.Component<Props, State> {
  constructor(props: Props) {
      super(props);
      this.state = {
          isOpen: false,
      };
}

render() {
    return (
        <div className={styles.cartContainer}>
            <button className={styles.button}
            onClick = {()=>{
                this.setState({isOpen: !this.state.isOpen})
            }}
            >购物车 2 (件)</button> 
            <div className={styles.cartDropDown} 
            style={{
                display: this.state.isOpen? "block" : "none"
            }}>

                <ul>
                    <li>robot 1</li>
                    <li>robot 2</li>
                </ul>
            </div>
        </div>
    )
  }
}

      State和Props

    • state
      • state是私有的，可以认为state是组件的私有属性。
      • 需要用setState()修改state。
        • 如果直接修改state，组件是不会触发render函数，页面不会重新渲染。
          //错误
this.state.isOpen = true

//正确
onClick = {()=>{
    this.setState({isOpen: !this.state.isOpen})
}}
      • 初始化state: 构建函数constructor
        //初始化state
constructor(props: Props) {
  super(props);
  this.state = {
      isOpen: false,
  };
}
      • state的更新是异步的。
        • 调用setState后，state不会立刻改变，是异步操作。
          • React会优化真正的执行时机。可能会将多个setState的方法合并为一次。
        • 不能依赖当前的state来计算下一个state。
          • 真正执行时候，this.state并不一定包含最新的变化。
    • props
      • 准确地说，是父组件传递向子组件的数据。
      • props数据是只读的，immutable。
        • 对象一旦创建就不可改变，只能通过销毁、重建来改变数据。
        • 通过判断内存地址是否一致，来确认对象是否有经过修改。
    • 区别： props是对外的，用于组件间的数据传递。state是对内的，用于组件内部的数据传递。

React Event 事件处理

  • 事件类型，将鼠标放在事件上会有提示，如onClick。
    • 但是显示出来的类型并不能直接使用，会报错，如:
    • 修改为以下类型就是可以。
      handleClick = (e: React.MouseEvent<HTMLButtonElement, MouseEvent>) => {
    this.setState({ isOpen: !this.state.isOpen })
}
    • e 中有两个子属性。
      • e.target描述事件发生的元素。
      • e.currentTarget描述事件处理绑定的元素。
      • 案例理解：
        • 比如绑定了onClick的buttonzhong ，有购物车icon和span包裹的文字两个元素。
        • e.target就会具体对应到用户是点击了哪个元素。
        • e.currentTarget则始终显示绑定了onClick的button。
      • 如果我们需要明确哪个元素(icon或者span文本)被点击，就需要做一点类型转换。
        • icon是使用 npm install react-icons安装的包。
        • 只有在点击span文本的时候，购物车才会弹出。
          handleClick = (e: React.MouseEvent<HTMLButtonElement, MouseEvent>) => {
  console.log("e.target", e.target);
  console.log("e.currentTarget", e.currentTarget);
  if ((e.target as HTMLElement).nodeName === "SPAN") {
      this.setState({ isOpen: !this.state.isOpen })
  }
}

React异步处理

  • 避免回调地狱(callback hell)
  • fake API数据：https://jsonplaceholder.typicode.com/
  • 如何获取并渲染？
    • 需要在类组件中使用，这边把原本是函数组件的App.tsx修改为类组件。因为类组件才有生命周期。
      1. 修改function为class，并继承React.Component。
      2. 将原本的return用render函数包裹起来。
      3. 使用constructor继承props和定义state
      4. 修改后的类组件如下：
        interface Props { }

interface State {
  robotGallery: any[]; 
}

class App extends React.Component<Props, State> {
  constructor(props: Props) {
    super(props);
    this.state = {
      robotGallery: [],
    };
  }

  render() {
    return (
      <div className={styles.app}>
        <div className={styles.appHeader}>
          <img src={logo} className={styles.appLogo} alt="logo" />
          <h1>Robot Platform</h1>
        </div>
        <ShoppingCart />
        <div className={styles.robotList}>
          {robots.map((r) => (
            <Robot id={r.id} email={r.email} name={r.name} />
          ))}
        </div>
      </div>
    );
  }
}
  • 在组件中调用生命周期componentDidMount()，并使用Fetch API获取异步数据。
    componentDidMount(){
  fetch("https://jsonplaceholder.typicode.com/users")
  .then((response) => response.json())
  .then((data) => this.setState({robotGallery: data}));
}
    • 返回的是一个promise
